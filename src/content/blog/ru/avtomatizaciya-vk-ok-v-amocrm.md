*Если сообщения от клиентов прилетают в пяти мессенджерах — твой сейлз никогда их все не закроет. Нужна воронка. Воронка — это CRM. А вот как в CRM загнать ВК и Одноклассники — вопрос отдельный.*

Сегодня прилетел вопрос от Евгения из бизнес-клуба. У него три личных аккаунта ВКонтакте, через которые идёт общение с клиентами. Чтобы сообщения попадали в amoCRM и менеджеры отвечали оттуда — сейчас платит **32 тысячи в год за каждый аккаунт**. Итого 96 тысяч.

Ещё он хочет подключить Одноклассники — но там, говорит, **готовых решений вообще нет**. При этом у него уже работает софт для рассылки в ОК с личных страниц, то есть технически общаться с ОК он умеет. Вопрос: можно ли как-то эти сообщения гнать в amoCRM?

Разбирал вечер. Рассказываю по пунктам.

#### 🔷 ВК — есть и сильно дешевле

Существует сервис **Pact.im** — подключает твои личные страницы ВК прямо к amoCRM: входящие летят в CRM, отвечаешь оттуда же, фото и видео ходят в обе стороны.

Цена — **около 900 ₽ в месяц за один аккаунт**. Итого за три аккаунта — **32 400 ₽ в год вместо текущих 96 000**. Экономия 64 тысячи в год, функционал тот же. Триал три дня — можно просто попробовать, ничего серьёзно не настраивая.

Подключение обычно **15–30 минут**: регистрируешься, авторизуешь ВК, связываешь с amoCRM. Самому разобраться реально.

Альтернатива — **i2crm**, тоже подтверждённо работает с ВК-личками, но вдвое дороже (~72 тысячи за три аккаунта в год). Wazzup24, Radist и Umnico — **только группы ВК**, для личных страниц не подходят.

Вывод: Евгений сейчас переплачивает примерно втрое. Pact.im решает.

#### 🔷 Одноклассники — готового действительно нет

Я прочесал рынок. **Ни один сервис** — ни для amoCRM, ни для любой другой CRM — **не умеет работать с ЛИЧНЫМИ страницами ОК**. Всё, что предлагается — только группы.

Причина в том, как устроены сами Одноклассники. У них есть официальный API только для групп (Bot API). Для личных страниц официального способа нет — его надо изобретать отдельно.

**Но это не значит, что задача нерешаемая.** Раз у Евгения уже работает рассыльщик по ОК с личных страниц — значит, способ общаться с ОК-личками технически существует. Обычно такие софты работают либо через эмуляцию браузера (headless Chrome), либо через реверс мобильного API ОК. Тот же механизм можно использовать, чтобы **читать** входящие, а не только слать.

Архитектура простая:

1. Свой маленький сервер-парсер читает входящие сообщения из ОК (через ту же лазейку, что и рассыльщик).
2. Отдаёт их в amoCRM через **amoCRM Chats API** — это официальный механизм самой amoCRM для создания «кастомных каналов» мессенджеров внутри CRM.
3. Менеджер отвечает в amoCRM, оттуда прилетает webhook на сервер-парсер, и он отправляет ответ обратно в ОК.

То есть amoCRM Chats API готов и ждёт — вопрос только в том, чтобы написать кусок между ним и ОК.

#### 🔷 Сколько это по времени

Честно, если смотреть как на отдельный проект:

- **Первоначальная разработка** — 2–4 недели работы специалиста. С нуля до работающего моста «ОК ↔ amoCRM».
- **Поддержка** — Одноклассники раз в 2–3 месяца что-то меняют в интерфейсе или мобильном приложении, интеграция после этого ломается. Починить — несколько часов разработчика за раз.

Плюс нужно помнить **риск блокировки**. Правила ОК запрещают автоматизацию — для ответов на входящие риск небольшой, для массовых рассылок высокий. ВК в этом смысле терпимее: там сервисы вроде Pact годами работают официально, никого не банят. В ОК такой «прослойки» на рынке нет — ты один на один с их правилами.

#### 🔷 Что бы я сделал на месте Евгения

**Шаг 1. По ВК — прямо сегодня перейти на Pact.im.** Триал 3 дня, подключение полчаса, экономия 64 тысячи в год. Никаких аргументов против — просто делать.

**Шаг 2. По ОК — сначала подумать, а можно ли вести общение с клиентами в группе, а не с личек.** Группа подключается к amoCRM легально через Umnico или Radist — дёшево, стабильно, рисков нет. Если сценарий бизнеса позволяет — это лучший путь. Многие не хотят потому что «привыкли с личного», но у клиента разницы никакой — он всё равно пишет в мессенджер.

**Шаг 3. Если общение именно через личные страницы критично** — тогда это индивидуальная разработка. И нужно честно прикинуть: объём сообщений в ОК оправдывает полтора-два месяца разработки плюс постоянные риски блокировки? Если нет — не ввязываться. Если да — это уже отдельный проект со своим бюджетом.

#### 🔷 Итого

- **ВК-лички + amoCRM** — решается готовым Pact.im, втрое дешевле текущего варианта.
- **ОК-группы + amoCRM** — решается готовыми Umnico или Radist, легально и дёшево.
- **ОК-лички + amoCRM** — только индивидуальная разработка через amoCRM Chats API. Технически реально, но взвешивай цену.
